================================================================================
SHARED PATTERNS - MUST FOLLOW EXACTLY
================================================================================

1. STATE MANAGEMENT
   - All UI state extends UiState interface
   - Use data classes with copy() for immutability
   - Computed properties allowed (val filteredDiaries: List = ...)
   - Example:
     data class MyUiState(
         val items: List<Item> = emptyList(),
         val selectedId: Int? = null,
         val loadStatus: LoadStatus = LoadStatus.INITIAL
     ) : UiState

2. VIEWMODEL PATTERN
   - Extend BaseViewModel<UiState, UiEvent>
   - Use setState { copy(...) } to update state
   - Use sendEvent(event) to send user feedback
   - All async work in viewModelScope.launch
   - Use Dispatchers.IO for DB/network
   - Catch exceptions and send error events
   - Example:
     @HiltViewModel
     class MyViewModel @Inject constructor(
         private val repo: MyRepository
     ) : BaseViewModel<MyUiState, MyUiEvent>(MyUiState()) {
         fun doSomething() {
             viewModelScope.launch(Dispatchers.IO) {
                 repo.getData().collect { data ->
                     setState { copy(items = data) }
                 }
             }
         }
     }

3. COMPOSABLE PATTERN
   - Inject ViewModel with @hiltViewModel()
   - Collect state with collectAsState()
   - Observe events with LaunchedEffect
   - Example:
     @Composable
     fun MyPage(viewModel: MyViewModel = hiltViewModel()) {
         val state by viewModel.state.collectAsState()
         LaunchedEffect(Unit) {
             viewModel.event.collect { event ->
                 when (event) {
                     is MyEvent.Success -> showSuccess()
                     is MyEvent.Error -> showError(event.msg)
                 }
             }
         }
         MyContent(state)
     }

4. NAVIGATION PATTERN
   - Use RootDestination.createRoute() helpers
   - Keep navigation simple (no complex params)
   - Use SavedStateHandle for ID params
   - Example:
     navController.navigate(RootDestination.createDiaryDetailRoute(diaryId))

5. ERROR HANDLING
   - Always use .catch { } blocks
   - Log errors with Log.e(TAG, "message", exception)
   - Send error events to UI
   - Show snackbars/dialogs to users
   - Never crash silently

6. MATERIAL 3 COMPONENTS
   - Use Material 3 components:
     * MaterialTheme.colorScheme (colors)
     * MaterialTheme.typography (fonts)
     * Material3 composables (Button, Card, etc.)
   - Use theme colors:
     * cs.background, cs.surface, cs.onBackground, etc.
   - Example:
     Text(
         text = "Hello",
         style = MaterialTheme.typography.titleMedium,
         color = MaterialTheme.colorScheme.onSurface
     )

================================================================================
KEY CLASSES & INTERFACES TO UNDERSTAND
================================================================================

BaseViewModel<S : UiState, E : UiEvent>
  - Protected: setState(reducer: S.() -> S)
  - Protected: sendEvent(event: E)
  - Public: state: StateFlow<S>
  - Public: event: SharedFlow<E>

LoadStatus (enum)
  - INITIAL, LOADING, SUCCESS, FAILURE

Diary (data class)
  - id: Int
  - title: String
  - content: String
  - createdDate: Date
  - updatedDate: Date

DiaryRepository (interface)
  - getDiaries(limit, offset): Flow<List<Diary>>
  - getDiaryById(id): Flow<Diary>
  - getDiaryByDate(date): Flow<Diary>
  - insertDiary(diary): suspend
  - updateDiary(diary): suspend
  - deleteDiary(diary): suspend
